# Weekly Sprint Rhythm

## Purpose
Maintain fast feedback loops with low coordination overhead.

## Entry Criteria
- Sprint scope is defined from validated work items.
- Open risks are listed in the sprint pre-mortem.

## Procedure
- Monday: plan sprint goals, commit risk register, lock top priorities.
- Wednesday: risk check + unblock decisions.
- Friday: demo shipped outcomes and finalize sprint report.
- Sunday: retrospective and next sprint prep.

## Exit Criteria
- Sprint report is signed off.
- Scorecard is updated with current week metrics.

## Outputs
- Work items with DoD evidence.
- Sprint report with metrics and retro actions.
- Updated weekly scorecard.

## Success Standard
No item is marked done without proof (`tests + docs + changelog + demo`).
