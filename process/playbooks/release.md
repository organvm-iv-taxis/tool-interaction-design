# Release Playbook

## Purpose
Promote changes safely with explicit go/no-go criteria.

## Entry Criteria
- Release candidate branch is ready.
- `release-readiness` artifact drafted.

## Procedure
1. Verify all release gates with evidence.
2. Confirm docs, changelog, and rollout notes.
3. Confirm rollback owner and tested strategy.
4. Make release decision (`approved`/`blocked`/`deferred`).
5. Publish and monitor post-release signals.

## Exit Criteria
- Decision recorded with rationale.
- Monitoring and rollback paths are active.

## Outputs
- Finalized release-readiness artifact.
