# Design Playbook

## Purpose
Make implementation decisions explicit before coding.

## Entry Criteria
- Intake artifact exists.
- Unknowns and risks are listed.

## Procedure
1. Describe target behavior and non-goals.
2. Define interfaces/contracts impacted.
3. Enumerate edge cases and failure modes.
4. Choose validation strategy (tests + checks).
5. Record release and rollback considerations.

## Exit Criteria
- Design decisions are documented.
- Test plan and rollback path are defined.

## Outputs
- Design note linked from the work item.
