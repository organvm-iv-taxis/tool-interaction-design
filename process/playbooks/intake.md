# Intake Playbook

## Purpose
Convert ideas into scoped, testable work items.

## Entry Criteria
- Problem statement exists.
- Intended user or operator is identified.

## Procedure
1. State the problem in one sentence.
2. Define success metrics and failure signals.
3. Create or update a `work-item` artifact.
4. Assign owner, phase, risk level, and estimate.
5. Add acceptance criteria and dependencies.

## Exit Criteria
- Work item validates against `work-item.schema.json`.
- Scope and owner are explicit.

## Outputs
- Updated `process/templates/work-item.example.yaml`-style artifact.
