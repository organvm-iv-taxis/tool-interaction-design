# Implementation Playbook

## Purpose
Ship changes in atomic, reviewable increments.

## Entry Criteria
- Design is approved.
- Branch and task scope are set.

## Procedure
1. Implement smallest viable slice first.
2. Keep commits atomic by concern.
3. Update contracts/templates/docs with code changes.
4. Run local quality checks before each push.
5. Track blockers directly in the work item.

## Exit Criteria
- Code, docs, and contracts are coherent.
- Local checks pass.

## Outputs
- PR-ready branch with linked evidence.
