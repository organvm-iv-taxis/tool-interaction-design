# 90-Day Implementation Roadmap

## Goal
Convert high-powered orchestration into disciplined, repeatable software delivery with lower cognitive load.

## Weeks 1-2 (Evaluation)
- [ ] Baseline audit of workflows, CI friction, and recurring failure patterns.
- [ ] Canonical lifecycle map: Discover -> Design -> Build -> Verify -> Release -> Operate.
- [ ] Gap ledger of missing SOPs, unclear ownership, and weak gates.

## Weeks 3-6 (Reinforcement)
- [ ] Publish SOP pack and enforce short-form execution steps.
- [ ] Enforce Definition of Done + demo evidence for each completed item.
- [ ] Adopt weekly sprint rhythm: plan, risk check, demo, retro.
- [ ] Consolidate quality checks under one command surface.

## Weeks 7-10 (Risk Analysis)
- [ ] Require pre-mortems for each sprint and release candidate.
- [ ] Enforce shatter-point controls (schema, migration, policy, observability).
- [ ] Run recurring recovery drills and measure recovery time.

## Weeks 11-13 (Growth)
- [x] Package process assets into reusable operator kits.
- [x] Run services lane (implementation/governance hardening).
- [x] Run products lane (templates/checklists/policy packs).
- [x] Run advisory lane (operating-system coaching with KPI reporting).

## KPI Targets
- Lead time down by 25% from baseline.
- Carry-over ratio under 0.20 by Week 8.
- Escaped defects near zero for two consecutive sprints.
- At least three monetizable assets shipped by Day 90.
